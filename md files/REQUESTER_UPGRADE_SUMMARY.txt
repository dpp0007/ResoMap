WHAT WAS UPGRADED
================================================================================

The Requester Dashboard has been transformed from a READ-ONLY request submission
interface into a FULLY FUNCTIONAL request management system.

BEFORE (Read-Only):
  ✗ Requesters could only create and view requests
  ✗ No ability to modify requests after creation
  ✗ No ability to cancel requests
  ✗ No communication channel with volunteers
  ✗ No feedback mechanism
  ✗ Limited dashboard visibility

AFTER (Fully Functional):
  ✅ Requesters can EDIT their own requests (when PENDING)
  ✅ Requesters can CANCEL their own requests (when PENDING/ASSIGNED)
  ✅ Requesters can ADD NOTES to requests for communication
  ✅ Requesters can SUBMIT FEEDBACK on completed requests (1-5 rating)
  ✅ Dashboard shows REQUESTER-SPECIFIC METRICS
  ✅ Full ACTION BUTTONS with MODALS for all operations
  ✅ Complete SERVER-SIDE VALIDATION and AUTHORIZATION
  ✅ Comprehensive AUDIT LOGGING

================================================================================
NEW REQUESTER CAPABILITIES (4 MAJOR FEATURES)
================================================================================

1. EDIT OWN REQUEST
   ├─ Availability: When status = PENDING only
   ├─ Editable Fields: Description (1-500 chars), Urgency Level
   ├─ Backend: RequesterActionServlet → RequestService → RequestDAO
   ├─ UI: Edit button + Modal form
   └─ Database: Updates description and urgencyLevel

2. CANCEL OWN REQUEST
   ├─ Availability: When status = PENDING or ASSIGNED
   ├─ Behavior: Soft cancel (status = CANCELLED), optional reason
   ├─ Backend: RequesterActionServlet → RequestService → RequestDAO
   ├─ UI: Cancel button + Modal form with reason field
   └─ Database: Sets status to CANCELLED, appends reason

3. ADD NOTES TO REQUEST
   ├─ Availability: For any active request (not COMPLETED/CANCELLED)
   ├─ Behavior: Append-only, timestamped, visible to Admin & Volunteer
   ├─ Backend: RequesterActionServlet → RequestService → RequestDAO
   ├─ UI: Add Note button + Modal textarea
   └─ Database: Appends note to request description

4. SUBMIT FEEDBACK ON COMPLETED REQUEST
   ├─ Availability: When status = COMPLETED only
   ├─ Fields: Rating (1-5, required), Comment (optional)
   ├─ Behavior: One feedback per request, timestamped
   ├─ Backend: RequesterActionServlet → RequestService → RequestDAO
   ├─ UI: Give Feedback button + Modal with radio buttons
   └─ Database: Appends feedback to request description
